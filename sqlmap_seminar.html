Concept

What is SQL Injection?

A web application becomes vulnerable when untrusted input changes the SQL query sent to the database. Instead of remaining only data, the input influences query logic, filtering, or execution.

  • Authentication bypass
  • Unauthorized data access
  • Data leakage or modification
mysql>
Simple vulnerable idea
SELECT * FROM users
WHERE username = '<input>'
AND password = '<input>';
If the application builds this query unsafely, attacker-controlled input may change the intended SQL behavior.
Anugrah K | Reg No: 12314674
Keep this slide conceptual. Do not overcomplicate syntax. The audience only needs the core mental model: unsafe input reaches a SQL query and changes behavior.

Classification

Types of SQL Injection

In-Band (Classic)
Error-based
Forces SQL errors that reveal database information in the page response.
UNION-based
Appends UNION SELECT to combine attacker queries with the original result set.
Inferential (Blind)
Boolean-based
Sends true/false conditions and observes content differences in the response.
Time-based
Injects SLEEP or BENCHMARK and measures response delay to infer data.
Advanced
Stacked queries
Terminates the original query and appends entirely new SQL statements.
Out-of-band (OOB)
Triggers DNS or HTTP requests to an external server controlled by the tester.

SQLmap automatically tests all applicable techniques against each parameter.

Tool overview

What is SQLmap?

SQLmap is an open-source penetration testing tool that automates SQL injection detection and, after confirmation, supports database enumeration in authorized environments.

Critical mindset

Reality Check: Not Every Website Is Vulnerable

Wrong expectation
anugrah@kali:~
anugrah@kali:~# sqlmap -u "https://abcd.com"

A random homepage without a testable parameter gives SQLmap little context. No useful input point means weak or no SQLi test path.

Better starting point
https://site.com/item.php?id=1
https://site.com/listproducts.php?cat=1
https://site.com/view.php?product=5
https://site.com/search.php?search=laptop

These parameters are not automatically vulnerable, but they are visible places where user input may enter a backend query.

Anugrah K | Reg No: 12314674
This is one of the most important slides. State directly that SQLmap does not dump random sites. A tester must first identify a candidate parameter or request field worth testing.

Practical step 1

Command 1: Detect Vulnerability

anugrah@kali:~
anugrah@kali:~# sqlmap -u "http://testphp.vulnweb.com/listproducts.php?cat=1"
[22:01:12] [INFO] testing connection to the target URL
[22:01:14] [INFO] testing if the target URL content is stable
[22:01:16] [INFO] heuristic (basic) test shows that GET parameter 'cat' might be injectable
GET parameter 'cat' is vulnerable.
[22:01:22] [INFO] the back-end DBMS is MySQL
sqlmap
The tool
-u
Target URL flag
"http://...php"
Page with input
?cat=1
Parameter to test
What it does
The -u flag tells SQLmap which URL to target. It parses the parameters from the query string automatically.
Why this command
Detection comes first. The tool checks whether cat behaves like an injectable input.
What to expect
Prompts, heuristics, payload testing, and either a confirmation or rejection.
Anugrah K | Reg No: 12314674
Read the command slowly. Explain each part: sqlmap, -u, full URL, and the parameter. Clarify that this is the first real command because it validates whether further steps make sense.

Parameter meaning

Why cat=1?

cat = parameter name
1 = value currently sent by the browser
It is not random. The application may use it to choose a category, product set, or database row.
mysql>
SELECT * FROM products
WHERE category_id = 1;
What SQLmap does with that value
Change parameter value
Observe response, errors, timing, or content changes
Decide whether behavior suggests injection
Only then move to deeper steps
Anugrah K | Reg No: 12314674
Remove confusion around cat=1. Tell them the parameter name is simply a field label, and the numeric value is the current input. SQLmap mutates this value to test how the application reacts.

Practical step 2

Command 2: Enumerate Databases

anugrah@kali:~
anugrah@kali:~# sqlmap -u "http://testphp.vulnweb.com/listproducts.php?cat=1" --dbs
[22:05:01] [INFO] retrieving database directory entries
available databases [2]:
[*] acuart
[*] information_schema
What it does
Lists database names accessible through the confirmed injectable path.
Why this command
It is the first enumeration step after successful detection.
What to expect
Database names such as application databases, system schemas, or test data stores.

Key teaching point: --dbs is not step one. It only becomes meaningful after injection is actually confirmed.

Operational realism

Limitations

WAF and filtering
Requests may be blocked, normalized, or rate-limited before SQLmap gets reliable feedback.
Authentication barriers
Some targets require valid sessions, tokens, or application state.
False positives and noise
A suspicious response does not always mean exploitable SQL injection.
No guaranteed vulnerability
Modern applications may correctly sanitize input and use prepared statements.
Anugrah K | Reg No: 12314674
Reinforce realism. Tools have constraints, and properly built applications resist this workflow. This helps students avoid exaggerated assumptions.

Defense first

Prevention

Parameterized queries
Keep code and data separate.
Prepared statements
Prevent user input from rewriting SQL logic.
Input validation
Reject malformed or unexpected data early.
Least privilege
Limit database account permissions.
Secure coding
Use safe database libraries and review patterns.
Regular testing
Validate fixes through reviews and security assessments.
Anugrah K | Reg No: 12314674
This slide should end the exploitation discussion and pivot strongly toward secure development and defense. Mention that prevention is more valuable than post-incident cleanup.

Closing

Conclusion

SQL injection is a serious web vulnerability, but detection starts with finding a meaningful input point.
A parameter such as cat=1 is simply a testable input that may influence backend SQL.
SQLmap automates detection and enumeration efficiently only after a candidate parameter is supplied and a vulnerability is confirmed.
Ethical authorization, scope discipline, and prevention remain central to professional penetration testing.
